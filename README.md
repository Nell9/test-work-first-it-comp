# Веб-сервис для учёта движения денежных средств (ДДС)

Небольшое веб-приложение для ведения учёта поступлений и списаний денег.
Можно создавать записи, редактировать и удалять их, фильтровать список и
управлять справочниками (статусы, типы, категории, подкатегории) с учётом
связей между ними.

Сделано на Django + Django REST Framework, фронтенд — обычные шаблоны с
Bootstrap и немного ванильного JS, который ходит в REST API.

## Что умеет

- Создание/редактирование/удаление записей ДДС.
- Список записей с фильтрами по периоду дат, статусу, типу, категории и
  подкатегории.
- Управление справочниками: статусы, типы, категории и подкатегории.
- Связи между сущностями:
  - категория привязана к типу;
  - подкатегория привязана к категории.
- В форме записи подкатегории подгружаются под выбранную категорию, а
  категории — под выбранный тип. Выбрать «несовместимую» пару нельзя.
- Валидация и на клиенте, и на сервере. Обязательные поля: сумма, тип,
  категория, подкатегория. Дата по умолчанию — сегодняшняя, но её можно
  поменять. Статус и комментарий — необязательные.

## Стек

- Python 3.11
- Django 5.1
- Django REST Framework 3.15
- django-filter (фильтрация списка)
- SQLite (база по умолчанию)
- Bootstrap 5 (подключается по CDN)

## Как запустить

Дальше команды для macOS/Linux. На Windows отличается только активация
виртуального окружения (`.venv\Scripts\activate`).

### 1. Зависимости

```bash
# из корня проекта
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. База данных

База — SQLite, отдельно ничего поднимать не нужно. Применяем миграции:

```bash
python manage.py migrate
```

При желании можно сразу залить демо-данные из примеров ТЗ (статусы Бизнес/
Личное/Налог, типы Пополнение/Списание, категории Инфраструктура и Маркетинг
с подкатегориями):

```bash
python manage.py seed_demo
```

Команда идемпотентна — повторный запуск дубликатов не создаёт.

### 3. Запуск

```bash
python manage.py runserver
```

Приложение будет доступно на http://127.0.0.1:8000/

Основные страницы:

- `/` — список записей с фильтрами;
- `/records/new/` — создание записи;
- `/records/<id>/edit/` — редактирование;
- `/references/` — управление справочниками.

### Админка (опционально)

Все модели зарегистрированы в стандартной админке Django. Чтобы зайти,
создайте суперпользователя:

```bash
python manage.py createsuperuser
```

И откройте `/admin/`.

## REST API

Фронтенд работает через REST API, его же можно дёргать напрямую. Корень — `/api/`.

| Метод | URL | Назначение |
|-------|-----|------------|
| GET/POST | `/api/records/` | список / создание записей |
| GET/PUT/PATCH/DELETE | `/api/records/{id}/` | одна запись |
| GET/POST | `/api/statuses/` | статусы |
| GET/POST | `/api/types/` | типы |
| GET/POST | `/api/categories/` | категории |
| GET/POST | `/api/subcategories/` | подкатегории |

Полезные параметры запроса:

- Записи: `?date_from=2025-01-01&date_to=2025-12-31&status=1&type=2&category=3&subcategory=4`
- Категории конкретного типа: `/api/categories/?type=2`
- Подкатегории конкретной категории: `/api/subcategories/?category=3`

Доступен и браузерный интерфейс DRF — просто откройте любой из адресов выше в
браузере.

## Тесты

```bash
python manage.py test
```

Покрыты модели и ограничения БД, CRUD по записям через API, валидация
обязательных полей и бизнес-правил (категория должна соответствовать типу,
подкатегория — категории), фильтрация и загрузка страниц.

## Пара слов про решения

- Статусы, типы, категории и подкатегории вынесены в отдельные таблицы, а не
  захардкожены — по ТЗ списки должны расширяться.
- Категория уникальна в рамках типа, подкатегория — в рамках категории.
  Поэтому, например, одноимённые категории в разных типах допустимы.
- Удаление элемента справочника, который ещё используется (есть записи ДДС
  или вложенные категории/подкатегории), блокируется (`on_delete=PROTECT`) —
  сервис вернёт понятное сообщение, а не молча потеряет данные. Чтобы удалить
  тип, нужно сначала убрать его категории и подкатегории. Неиспользуемые
  элементы удаляются свободно.
- Аутентификацию специально не делал — в ТЗ её нет, сервис открытый.
